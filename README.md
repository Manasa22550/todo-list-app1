# Interactive To-Do List Application

## Overview
This project is an interactive, web-based To-Do List application built as part of Task 2 of the InternSpark Web Development Internship. The core focus of this task is implementing clean Document Object Model (DOM) manipulation using structural HTML, stylish responsive CSS, and native Vanilla JavaScript logic to enable dynamic client-side interactions.

## Project Objectives
* **Dynamic DOM Manipulation:** Programmatically create, modify, append, and destroy structural HTML element nodes dynamically based on real-time user inputs.
* **Responsive Architecture:** Maintain fluid mobile, tablet, and desktop layout presentation fields using CSS modern Flexbox techniques.
* **Input Validation:** Implement safety gates preventing empty task entry logs.
* **User Experience Enhancements:** Apply keypress event structural listening maps to allow swift keyboard operations alongside standard visual click interactions.

## Core Features
* **Dynamic Element Addition:** Adds user text values into distinct list items immediately upon form submission execution.
* **Single Task Destruction:** Every list element carries a self-destruct trigger button to prune items in real time.
* **Keyboard Navigation Integration:** Features full tracking mapping for the `Enter` keyboard trigger inside text field focuses.
* **Fluid UI Transitions:** Incorporates custom translation and opacity keyframe styling routines to smooth out screen layout updating.

## Technologies Used
* **Structure:** HTML5 Semantic elements
* **Presentation:** CSS3 layout formatting blocks and keyframe animations
* **Logic Brain:** JavaScript (ES6+ Native Functions & DOM Listening nodes)
* **Environment Tools:** Visual Studio Code & GitHub Desktop
* **Cloud Hosting Platform:** GitHub Pages Production Framework

## File Structure Layout
* `index.html` — Holds structural containers, form interaction fields, and injection targets.
* `style.css` — Orchestrates linear gradient background canvases, central grid boundaries, and animation maps.
* `script.js` — Controls core DOM functional listeners, element node creation pipelines, and target removal handlers.
